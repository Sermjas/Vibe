# Рекомендуемая схема безопасности mcp.json

Короткая шпаргалка для безопасной работы MCP GitHub на нескольких устройствах.

## Базовая схема

- Не хранить токен в `mcp.json`, `.env` в репозитории и других синхронизируемых файлах.
- Хранить токен в переменной окружения ОС: `GITHUB_PERSONAL_ACCESS_TOKEN`.
- Держать сам токен в менеджере паролей (1Password, Bitwarden, KeePass).
- По возможности использовать отдельный токен для каждого устройства.
- Выдавать токену минимально необходимые права и срок действия.

## Шаблон mcp.json (без секрета)

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-github"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_PERSONAL_ACCESS_TOKEN}"
      }
    }
  }
}
```

Если подстановка `${...}` не поддерживается вашей версией, удалите блок `env` и оставьте только переменную окружения в системе.

## Мини-чеклист "Новое устройство за 2 минуты"

1. Установить Cursor и открыть проект.
2. Скопировать токен из менеджера паролей.
3. Задать переменную окружения:
   - Windows (PowerShell): `setx GITHUB_PERSONAL_ACCESS_TOKEN "ghp_..."`
   - macOS/Linux (bash/zsh): `export GITHUB_PERSONAL_ACCESS_TOKEN="ghp_..."`
4. Перезапустить Cursor/терминал.
5. Проверить работу GitHub MCP (любой простой запрос).
6. Убедиться, что токен не записан в `mcp.json`.

## Ротация и безопасность

- При утечке сразу отозвать токен в GitHub и выпустить новый.
- Для контроля инцидентов использовать имена токенов по устройствам: `cursor-laptop`, `cursor-home-pc`.
- Периодически проверять активные токены и удалять неиспользуемые.
